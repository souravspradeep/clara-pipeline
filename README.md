# Clara Answers - Automation Pipeline

## What This Does
Converts call recordings into configured AI voice agent specs for Clara Answers.

## Architecture

Demo Call -> Whisper Transcription -> Groq LLM Extraction -> Account Memo (v1) -> Agent Spec (v1)
                                                                    |
Onboarding Call -> Extract Changes -> Patch v1 -> Account Memo (v2) -> Agent Spec (v2) + Changelog
                                                                    |
                                                         retell_server.py (Flask + Groq)
                                                                    |
                                                    Retell Custom LLM -> Live Voice Agent

## Quick Start

### 1. Install dependencies
pip install groq faster-whisper python-dotenv deepdiff flask requests

### 2. Set up environment
Copy .env.example to .env and add your keys:
GROQ_API_KEY=your_groq_key_here
OUTPUT_DIR=./outputs/accounts
DATA_DIR=./data
TRELLO_API_KEY=your_trello_key_here
TRELLO_TOKEN=your_trello_token_here
TRELLO_LIST_ID=your_trello_list_id_here

Get free Groq key at: https://console.groq.com
Get free Trello key at: https://trello.com/app-key

### 3. Add transcripts to data folder
data/demo/001_bens_electric_demo.txt
data/onboarding/001_bens_electric_onboard.txt

### 4. Update manifest
Edit data/manifest.json with your account names and file paths

### 5. Run single account

Demo call - creates v1:
python scripts/ingest.py --file data/demo/001_bens_electric_demo.txt --type demo

Onboarding call - creates v2:
python scripts/ingest.py --file data/onboarding/001_bens_electric_onboard.txt --type onboarding --account_id ben's_electric_solutions_team_20260304

### 6. Run all accounts at once
python scripts/batch_run.py

## Output Structure

outputs/accounts/{account_id}/
    v1/
        account_memo.json        <- extracted from demo call
        retell_agent_spec.json   <- Clara conversation script (v1)
    v2/
        account_memo.json        <- updated from onboarding call
        retell_agent_spec.json   <- Clara conversation script (v2)
        changes.json             <- what changed and why

changelog/
    {account_id}.md              <- human-readable change summary per account

## Retell Setup

### Why No Direct API Integration
Retell's free tier only supports Custom LLM mode — it does not allow
built-in LLM selection without a paid plan.

### How It Works
We run a local Flask server (retell_server.py) that receives messages
from Retell, calls Groq (free), and returns Clara's response.

Caller -> Retell -> retell_server.py (Flask + Groq) -> response -> Retell -> Caller

### Step 1: Start the Flask server
python retell_server.py

You should see:
 * Running on http://127.0.0.1:5000

### Step 2: Expose it publicly with ngrok
In a second terminal:
ngrok http 5000

Copy the forwarding URL shown, example:
https://abc123.ngrok-free.app

### Step 3: Connect to Retell
1. Go to https://retell.ai and log in
2. Click Create Agent -> Custom LLM
3. Paste your URL as: https://abc123.ngrok-free.app/llm
4. Click Publish

### Step 4: Test it
python test.py

You should get a JSON response with Clara's reply.

### Important Note
ngrok free tier gives a new URL every time you restart it.
Update the Custom LLM URL in Retell whenever you restart ngrok.

### What the system_prompt contains
The system prompt is auto-generated by the pipeline and embedded in
retell_server.py. It includes:

Business hours flow:
- Greeting -> ask purpose -> collect name and number
- Transfer call -> fallback if transfer fails
- Confirm next steps -> ask if anything else -> close

After hours flow:
- Greeting -> ask purpose -> confirm if emergency
- If emergency: collect name, number, address -> attempt transfer
- If transfer fails: apologize and assure quick follow-up
- If non-emergency: collect details -> confirm business hours follow-up
- Ask if anything else -> close

## Known Limitations
- ngrok free tier URL changes on every restart — must update Retell Custom LLM URL manually
- retell_server.py must be running locally for Retell agent to work
- Whisper transcription runs on local CPU (~5 mins per file)
- Groq free tier: 14,400 requests per day
- n8n Execute Command needs N8N_ALLOW_EXEC=true Docker flag
- Tested on 1 account only — 4 remaining accounts pending data
